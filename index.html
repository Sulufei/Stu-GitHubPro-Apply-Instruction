📋 申請前準備

學生申請資格與所需文件

  • 目前就讀大學、碩博士等學位課程（至少 13 歲）
  • 擁有學校核發的電子郵件帳號（@xxx.edu.tw）
  • 已建立個人 GitHub 帳號（免費即可）
  • 備妥以下任一證明文件（照片或掃描檔）：學生證（需顯示在學年度）、當學期課表截圖、在學證明書或成績單
⚠️
建議使用手機拍攝學生證等文件，畫素較高、審核通過率更高。文件需清晰可辨，不可有遮擋。

教職員申請資格與所需文件

  • 現職教師、研究員或學校職員
  • 擁有學校核發的電子郵件帳號（@xxx.edu.tw）
  • 已建立個人 GitHub 帳號（免費即可）
  • 備妥以下任一證明文件：教職員證或服務證、學校出具的任職證明函、近期薪資單（需顯示學校名稱）
ℹ️
教師審核通常需要 1–14 個工作天，視 GitHub 人工審核排程而定，建議提早申請。

📝 GitHub Education 申請流程

💡
學生與教師的申請流程相同，差異僅在選擇身分時，以下步驟適用於兩者。
📱
強烈建議：使用手機行動網路（4G/5G）申請！ 台灣許多大學透過 TANet（台灣學術網路）對外連線，出口 IP 位於台北，GitHub 的地理位置驗證機制可能因此誤判您不在校園內，導致申請被拒絕。

申請前請這樣做：
  1. 使用手機開啟瀏覽器（Chrome / Safari）
  2. 關閉 WiFi，改用行動數據（4G / 5G）
  3. 確認手機的位置存取權限已開啟
  4. 前往 https://github.com/education 進行申請
✅ 此方法已經本校電算中心實測確認，可有效避免位置驗證失敗被拒絕的問題。
  1. 建立或登入 GitHub 個人帳號

    前往 github.com，若尚未有帳號請先完成免費註冊。使用個人常用 email 建立帳號即可。

    🖼️ images/step-01-github-login.jpg 截圖內容：github.com 首頁，顯示「Sign in」與「Sign up」按鈕的畫面
    ▲ GitHub 首頁 / 登入頁面
  2. 將學校信箱加入 GitHub 帳號

    前往 Settings → Emails，點擊「Add email address」，輸入學校 email（帳號@stu.edu.tw）並完成信箱驗證（收信點連結）。

    🖼️ images/step-02-add-email.jpg 截圖內容：GitHub Settings → Emails 頁面，顯示「Add email address」欄位與新增按鈕
    ▲ Settings → Emails：新增並驗證學校信箱
    ⚠️
    請確認學校信箱已完成 Email 驗證（收信並點擊驗證連結），否則後續申請將無法通過。
  3. 前往 GitHub Education 申請頁

    前往 education.github.com，點擊頁面中的「Join GitHub Education」按鈕。

    🖼️ images/step-03-education-home.jpg 截圖內容：education.github.com 首頁，顯示醒目的「Join GitHub Education」按鈕
    ▲ GitHub Education 首頁
  4. 進入 Education Benefits 設定頁

    也可直接前往 github.com/settings/education/benefits，在「GitHub Education」區塊點擊「Start an application」。

    🖼️ images/step-04-settings-education.jpg 截圖內容：github.com/settings/education 頁面，顯示「Start an application」按鈕
    ▲ Settings → Education：開始申請
  5. 選擇身分並填寫表單

    選擇「Student（學生）」或「Teacher（教師）」，搜尋並選擇學校名稱、選擇學校信箱，填寫 GitHub 的使用用途。

    🖼️ images/step-05-select-role.jpg 截圖內容：申請表單頁面，顯示身分選擇（Student / Teacher）、學校名稱搜尋欄、信箱選擇下拉選單
    ▲ 選擇身分與填寫學校資訊
  6. 上傳身分證明文件

    點擊上傳區域，上傳學生證、課表或在學證明等照片，確保文字清晰可辨後，點擊「Submit application」送出。

    🖼️ images/step-06-upload-doc.jpg 截圖內容：文件上傳頁面，顯示上傳按鈕（或拖曳區域）、預覽照片，以及「Submit application」按鈕
    ▲ 上傳在學/在職證明文件並送出
  7. 【若出現】「Why are you not on campus?」錯誤處理

    上傳文件後，若出現如下錯誤提示，代表 GitHub 偵測到您的 IP 位置與學校地理位置不符。台灣許多大學透過 TANet（台灣學術網路）對外連線，出口 IP 位於台北，即使人在校園內也可能觸發此警告，屬正常現象。

    🖼️ images/step-06b-not-on-campus.jpg 截圖內容：出現「Your application cannot be reviewed until you fix the following: Please upload an image to help us understand why you are so far from campus.」錯誤提示的畫面
    ▲ TANet 連線觸發的校外 IP 警告畫面

    處理步驟

    1. 選擇 「Other」（因為是 TANet 網路問題，不屬於其他三個選項）
    2. 在 「Please provide a brief explanation」 欄位填入以下說明（可複製）：
      I am applying from a computer on campus. My university connects to the internet through TANet (Taiwan Academic Network), whose IP exit node is located in Taipei, which may appear geographically distant from our campus. I am a current student/faculty at [學校英文全名].
    3. 在圖片上傳區域補充上傳學生證照片或課表截圖作為佐證
    4. 點擊送出，等待人工審核（通常 1–5 個工作天）
    💡
    若您有使用 VPN，請先關閉 VPN 再重新申請，可直接略過此步驟。台灣學校透過 TANet 連線者，即使在校內申請也可能遇到此問題，填寫說明後人工審核通常均可通過。
  8. 【若出現】申請被拒絕（Rejected）的處理方式

    送出後若收到拒絕通知，頁面會顯示紅色「Rejected」狀態與原因清單。不必擔心，依照以下步驟修正後重新申請即可通過。以下為最常見的拒絕原因與解決方法：

    🖼️ images/step-07b-rejected.jpg 截圖內容：Education Benefits 頁面顯示紅色「Rejected」狀態與拒絕原因清單的畫面
    ▲ 申請被拒絕的頁面（Rejected）

    📋 常見拒絕原因對照表

    GitHub 顯示的拒絕原因 解決方法
    Please complete your GitHub billing information with your full name exactly as it appears in your academic affiliation document. 最關鍵：前往 Payment information 填寫與證件完全一致的真實姓名（見下方步驟）
    You are unlikely to be verified until you have completed your GitHub user profile with your full name. 前往 Settings → Public profile，將「Name」欄位改為與證件一致的真實姓名
    Please secure your GitHub account with two-factor authentication. 前往 Settings → Password and authentication，開啟雙重驗證（2FA）
    You must configure your browser and operating system to allow location access. You may not use a VPN. 關閉 VPN；或選擇 Other 並說明為 TANet 連線問題（參考上方步驟）

    🔧 最重要的修正步驟：填寫 Payment information

    前往 Settings → Billing and licensing → Payment information，依序填寫以下資訊：

    • First name / Last name：填寫與學生證、教職員證完全一致的真實姓名（英文）；若只有單一姓名，請在 First name 與 Last name 欄位填入相同內容
    • Address：No.59, Hengshan Rd., Yanchao Dist.，City 填: Kaohsiung
    • Country/Region：選擇 Taiwan
    • 填完後點擊「Save billing information」儲存
    🖼️ images/step-07c-payment-info.jpg 截圖內容：Settings → Billing and licensing → Payment information 頁面，顯示 First name、Last name、Address、City、Country/Region 等欄位與「Save billing information」按鈕
    ▲ Payment information：填寫與證件一致的姓名及學校地址
    ⚠️
    修正完所有項目後，請先登出再重新登入 GitHub，然後回到 github.com/settings/education/benefits 點擊「Start an application」重新送出申請。

    ✅ 重新申請成功：進入審查中狀態

    修正後重新送出，頁面應顯示綠色進度條與「Pending」狀態，表示申請已送出並進入人工審查，靜待 GitHub 通知信即可。

    🖼️ images/step-07d-pending.jpg 截圖內容：Education Benefits 頁面顯示綠色進度條與「Pending」狀態，顯示「Your application has been received and is currently pending review.」的畫面
    ▲ 重新申請成功：進入 Pending（審查中）狀態
  9. 等待審核結果

    送出後 GitHub 進行自動或人工審核。學生通常 數小時～3 天可得到結果；教師有時需 1–14 個工作天。

    ⚠️
    審核結果不一定會寄送通知信至學校信箱。建議定期回到 github.com/settings/education/benefits 頁面查看申請狀態。

    ✅ 審核通過：「Approved / Awaiting Benefits」畫面

    通過審核後，頁面將顯示綠色「Approved」狀態與以下重要資訊：

    • Your academic status has been verified. Congratulations!
    • 教育福利將在通過驗證後的 72 小時內正式生效
    • 點擊頁面中的 「link」 連結，可直接兌換 Copilot Pro 免費資格
    • 方案有效期限為 2 年（例：2026/3/19 通過 → 到期日為 2028/3/19）
    🖼️ images/step-09-approved.jpg 截圖內容：Education Benefits 頁面顯示綠色「Approved」狀態，內容包含「Your academic status has been verified. Congratulations!」、「after 72 hours」與到期日的畫面
    ▲ 審核通過：Approved / Awaiting Benefits 畫面

    ⏳ 等待 72 小時後：福利正式生效

    約 72 小時後，再次前往 github.com/settings/education/benefits，進度條會變成綠色 100%，狀態由「Awaiting Benefits」升級為「Verified (benefits available)」，並顯示「Coupon applied」與到期日。

    🖼️ images/step-09b-coupon-applied.jpg 截圖內容：Education Benefits 頁面顯示「Coupon applied」與綠色 100% 進度條，狀態為「Verified (benefits available)」，並顯示到期日 March 19, 2028 的畫面
    ▲ 72 小時後福利生效：Coupon applied，進度條 100%

💻 在 VS Code 中使用 Copilot Pro

  1. 安裝 Visual Studio Code

    若尚未安裝，請前往 code.visualstudio.com 下載對應作業系統版本（Windows / macOS / Linux）。

    🖼️ images/step-11-vscode-download.jpg 截圖內容：code.visualstudio.com 下載頁面，顯示各作業系統版本下載按鈕
    ▲ VS Code 官網下載頁面
  2. 安裝 GitHub Copilot 擴充套件

    開啟 VS Code，按 Ctrl+Shift+X（macOS：Cmd+Shift+X）開啟擴充套件面板，搜尋「GitHub Copilot」，安裝發行者為 GitHub 的官方套件。

    🖼️ images/step-12-install-extension.jpg 截圖內容：VS Code Extensions 面板搜尋「GitHub Copilot」，顯示官方套件（發行者 GitHub）與「Install」按鈕
    ▲ Extensions 面板安裝 GitHub Copilot 官方套件
    ℹ️
    安裝「GitHub Copilot」後，「GitHub Copilot Chat」通常會一併安裝。若未自動安裝，請手動搜尋安裝。
  3. 點擊登入 GitHub 帳號

    安裝完成後，VS Code 左側活動列底部帳號圖示（Accounts）點擊後，選擇「Sign in with GitHub to use GitHub Copilot」。

    🖼️ images/step-13-vscode-signin.jpg 截圖內容：VS Code 左下角帳號圖示展開後顯示「Sign in with GitHub to use GitHub Copilot」選項，或右下角狀態列 Copilot 圖示登入提示
    ▲ VS Code 帳號選單 → 登入 GitHub
  4. 瀏覽器跳出 GitHub 授權頁，選擇帳號並授權

    VS Code 將自動開啟瀏覽器，依序出現以下兩個畫面：

    【畫面一】選擇授權帳號：出現「Authorize Visual Studio Code」頁面，顯示目前已登入的 GitHub 帳號。確認帳號正確後，點擊「Continue」按鈕。若要改用其他帳號，點擊「Use a different account」。

    🖼️ images/step-14-github-authorize.jpg 截圖內容：瀏覽器顯示「Authorize Visual Studio Code」頁面，列出已登入的 GitHub 帳號（Signed in as xxx）與「Continue」按鈕，以及「Use a different account」選項
    ▲ 畫面一：選擇 GitHub 帳號，點擊「Continue」

    【畫面二】允許開啟 VS Code：瀏覽器跳出「要開啟 Visual Studio Code 嗎？」的確認對話框，並顯示「Launching Visual Studio Code」頁面。點擊「開啟 Visual Studio Code」返回編輯器。

    🖼️ images/step-14b-launch-vscode.jpg 截圖內容：瀏覽器顯示「Launching Visual Studio Code」頁面，同時跳出「要開啟 Visual Studio Code 嗎？」對話框，包含「開啟 Visual Studio Code」與「取消」按鈕
    ▲ 畫面二：點擊「開啟 Visual Studio Code」返回編輯器

❓ 常見問題 Q&A

🎓 Q1：學生申請後多久可以使用 Copilot Pro？
通過 GitHub Education 審核後，教育福利最多需 72 小時才會正式啟用。啟用後可收到學校信箱通知，前往 github.com/settings/copilot 即可開啟。整體流程最快當天完成，最慢約需 3 個工作天。
👩‍🏫 Q2：教師審核需要多久？
教師申請通常需要 1 至 14 個工作天不等，取決於 GitHub 的人工審核排程與文件清晰度。建議提交清晰的任職證明文件，並確保學校信箱已通過驗證，以加快審核速度。
📧 Q3：我沒有學校 email，可以申請嗎？
若沒有學校 email，仍可嘗試申請，但需上傳更完整的在學/任職證明文件（如在學證明書、教職員證照片）。有學校 email 並完成驗證，審核通過機率顯著較高。請聯繫電算中心確認是否可申請學校 email。
❌ Q4：申請被拒絕（Rejected）怎麼辦？
常見拒絕原因與解決方式：
  • 學校 email 未完成驗證 → 前往 Settings → Emails 補完驗證步驟
  • 文件照片不清晰 → 重新拍攝，確保文字清楚可辨
  • 文件顯示資訊不足 → 改用在學證明書或含有效日期的學生證
  • 學校名稱無法識別 → 補充上傳含學校抬頭的官方文件
可重新前往 github.com/settings/education/benefits 重新提交申請。
⏰ Q5：Copilot Pro 教育版會到期嗎？
是的。教育身分有效期通常為 2 年，或至預計畢業/離職日。到期前 GitHub 會提醒您重新驗證（Reverification）。只要仍具在學或在職資格，即可持續更新免費方案。
🔁 Q6：如何確認目前是 Copilot Pro 而非 Free？
前往 github.com/settings/copilot，若顯示「GitHub Copilot Pro — Free for Education」或「Active (Free)」，即表示已成功使用教育免費方案。
💻 Q7：除了 VS Code，Copilot Pro 支援哪些編輯器？
GitHub Copilot Pro 支援多種主流開發工具，包括：
  • Visual Studio Code（最常用、功能最完整）
  • Visual Studio 2022
  • JetBrains IDE（IntelliJ、PyCharm、WebStorm 等）
  • Neovim / Vim
  • GitHub.com 網頁版（Copilot Chat）
🔐 Q8：我的程式碼會被 GitHub 儲存或拿去訓練嗎？
根據 GitHub 官方說明，個人帳號的 Copilot 預設不會將程式碼用於訓練模型。可在 Settings → Copilot → Policies 頁面確認並關閉「Allow GitHub to use my code snippets for product improvements」選項，以進一步保護隱私。
🏫 Q9：學校需要事先與 GitHub 簽約才能申請嗎？
不需要。GitHub Education 計畫是 GitHub 提供給個人（學生/教師）的免費方案，由個人自行申請並提供身分證明，不需要學校統一簽約或授權。只要學校為合法立案的教育機構，師生均可個別申請。
